{% trans "Document Signing" %}

{% trans "Grantify offers two ways to collect signatures on award agreements. You can use either option depending on your agency's needs." %}

{% trans "Option A: Built-in Signing Flows (Recommended)" %}

{% trans "Native signing flows let you define a sequential chain of internal staff signers with full control over the process:" %}

  1. {% trans "Navigate to Awards → Signature Flows and create a flow (or use one already linked to your grant program)." %}
  2. {% trans "Add signing steps in order (e.g., Program Officer → Division Director → Commissioner). Each step can be assigned to a specific user or a role." %}
  3. {% trans "Upload the PDF award agreement as a document template." %}
  4. {% trans "Click Edit Placements to open the visual editor. Select a step, choose the field type (Signature, Initials, Date, or Name), and click on the PDF where it should appear." %}
  5. {% trans "From the award detail page, click Start Signing Flow and assign a signer to each step." %}
  6. {% trans "Each signer receives an email notification in sequence. They can sign using a typed name, an uploaded image, or by drawing with their mouse or finger." %}
  7. {% trans "Once all signers complete, a final signed PDF is generated automatically with all signatures overlaid at the defined positions. A full audit trail records who signed, when, and from which IP address." %}
{% trans "Option B: DocuSign Integration" %}

{% trans "For external signers or organizations that already use DocuSign:" %}

  1. {% trans "Navigate to the award detail page and click Request Signature (DocuSign)." %}
  2. {% trans "Enter the signer's name and email, optional CC, and notes." %}
  3. {% trans "Click Send for Signature. The system sends a DocuSign envelope via JWT-authenticated API." %}
  4. {% trans "Once signed, the document is automatically downloaded and the award status updates to Executed." %}
{% trans "Both options produce a signed document with a complete audit trail. Use built-in signing flows for internal staff sign-off chains (sequential approval workflows). Use DocuSign when you need external parties to sign or when your agency requires DocuSign specifically." %}
{% trans "Managing Signature Flows" %}

{% trans "Signature flows are reusable templates that define who needs to sign and in what order. Administrators can configure flows, while any authenticated user can define where on a document signatures should be placed." %}

{% trans "Configuring Flows" %}

{% trans "Go to Awards → Signature Flows to create and manage flows. Each flow can optionally be linked to a grant program so it appears automatically on awards for that program." %}

{% trans "Adding Steps" %}

{% trans "Add steps with a label (e.g., 'Program Officer Approval'), an execution order, and an assignment type. Steps assigned by role allow you to pick the specific signer when initiating a packet; steps assigned to a specific user are pre-assigned." %}

{% trans "PDF Templates & Placement Editor" %}

{% trans "Upload the PDF document that needs signatures. Then open the Placement Editor — a visual tool where you click on the PDF to define where each signer's signature, initials, date, or printed name should appear. Placements are stored as percentages, so they work at any zoom level or resolution." %}

{% trans "Tracking Packets" %}

{% trans "View all active and completed signing packets under Awards → Signing Packets. Each packet shows a progress bar, the current active signer, and a full timeline. You can send reminders to the active signer or cancel a packet if needed." %}

{% trans "My Signatures" %}
